# GIS · ВОЛС — установка: сервер + приложение Windows

Простая инструкция для администратора и инженеров.  
Функции программы — в [ДОКУМЕНТАЦИЯ.md](ДОКУМЕНТАЦИЯ.md).

---

## Как это работает (рекомендуемая схема)

```
┌─────────────────────┐         сеть          ┌──────────────────────────┐
│  ПК Windows         │   HTTP :4000          │  Сервер (офис / VM)      │
│  приложение         │ ───────────────────►  │  только BACKEND          │
│  «GIS VOLS»         │                       │  backend/data/*.sqlite   │
│  (Electron .exe)    │                       │  npm start               │
└─────────────────────┘                       └──────────────────────────┘
```

- На **сервере** — один раз поднимаете **backend** (API, базы, пользователи). Сайт в браузере на сервере **не обязателен**.
- На **каждом ПК Windows** — ставите **приложение GIS VOLS** (окно Electron). Оно подключается к серверу по адресу из файла настроек.
- Несколько инженеров в одной базе («юл-ком») — см. [совместная работа](ДОКУМЕНТАЦИЯ.md#121-совместная-работа-нескольких-пользователей).

---

## Что нужно заранее

| Где | Что |
|-----|-----|
| **Сервер** | Node.js 20 LTS, папка `backend`, порт **4000**, секрет `JWT_SECRET` |
| **ПК сборки** (один раз) | Node.js, папка `frontend`, для `.exe` — **Windows** (сборка установщика) |
| **ПК пользователей** | Windows 10/11, установщик `GIS-VOLS-Setup-….exe`, файл `gis-desktop.json` с IP сервера |
| **Бэкапы** | Копировать `backend/data/` на сервере |

# Часть 1. Сервер — только backend

Подходит Linux или Windows Server.

### 1. Node.js и проект

```bash
node -v    # v20.x или 18+
```

Скопируйте на сервер папку **`backend`** (достаточно только её, если все работают через desktop-приложение).

### 2. Установка и запуск

```bash
cd backend
npm install
```

Задайте секрет (обязательно в production):

```bash
# Linux
export JWT_SECRET="длинный-случайный-секрет"
export PORT=4000
npm start
```

```bat
REM Windows Server
cd C:\GIS\backend
set JWT_SECRET=длинный-случайный-секрет
set PORT=4000
npm start
```

В логе: `GIS backend listening on http://localhost:4000`.

### 3. Проверка

С любого ПК в сети в браузере: `http://IP_СЕРВЕРА:4000/health` → JSON `"ok": true`.

### 4. Файрвол

Откройте **TCP 4000** для офисной подсети (не в интернет без VPN и HTTPS).

#### macOS (MacBook как сервер)

1. **Узнайте IP Mac** (тот, что в Wi‑Fi, не `127.0.0.1`):

   ```bash
   ipconfig getifaddr en0
   ```

   Если пусто — `ifconfig | grep "inet "` и возьмите адрес вида `192.168.x.x`.

2. **Проверка на самом Mac** (backend должен быть запущен: `cd backend && npm start`):

   - `http://127.0.0.1:4000/health` — обязательно работает  
   - `http://ВАШ_IP:4000/health` — тот же JSON; если здесь **не** работает, а localhost работает — файрвол или неверный IP

3. **Системные настройки → Сеть → Брандмауэр** (Firewall):

   - Включён → **Параметры…** / **Options…**
   - Разрешите входящие для **node** или **Terminal** (или временно выключите брандмауэр для проверки)
   - Либо: **Параметры → Режим разработчика** не влияет на порт; нужно именно правило для Node

4. **Перезапуск backend** после обновления кода (слушает `HOST=0.0.0.0` по умолчанию — доступ из LAN):

   ```bash
   cd backend
   npm start
   ```

   В логе: `HOST=0.0.0.0` и подсказка `С другого ПК: http://<IP-этого-Mac>:4000/health`.

5. **С Windows**: в cmd `ping IP_MAC`, в браузере `http://IP_MAC:4000/health`. Только после этого настраивайте `gis-desktop.json` в папке с `GIS VOLS.exe`.

#### Карта и база по LAN (большой workspace)

Если на Mac **больше ~2000 узлов**, клиент работает в режиме **подгрузки по области** (не вся сеть сразу). По Wi‑Fi задержка **5–300 ms** на запрос — это нормально; после быстрого зума дождитесь **полоски загрузки** под шапкой на вкладке «Карта».

- **Линии** (канализация / ВОЛС) подгружаются на всех масштабах; **муфты и кроссы** на карте видны при **zoom ≥ 16** (при отдалении остаются только ТК — не ошибка).
- Не крутите колёсико зума подряд без паузы: клиент схлопывает запросы, но каждая область — несколько HTTP-запросов к Mac.
- Вкладка **«База данных»**: при первом открытии счётчик может показывать «—» и скелетон строк, пока идёт запрос (данные не «пропали»).
- После обновления кода на Mac: **перезапустите** `npm start` (на backend включено сжатие gzip). На Windows — **пересоберите** exe (`cd frontend && npm run build && npm run build:desktop:dir`) и скопируйте новый `win-unpacked`.
- Уведомления на карте (колокольчик): при удалённом API — подсказка «LAN»; фоновая синхронизация **не сбрасывает** весь кэш карты.

#### В приложении написано «Сервер API localhost»

Значит **`gis-desktop.json` не найден или не прочитан**. Проверьте:

1. Файл **в той же папке**, что `GIS VOLS.exe` (например `release\win-unpacked\`), имя **`gis-desktop.json`** (не `.txt`).
2. Содержимое (подставьте IP Mac):

```json
{
  "apiUrl": "http://192.168.1.177:4000"
}
```

3. Сохраните в **Блокноте** как «Все файлы» → UTF-8. Перезапустите приложение.

4. **Без пересборки** — запуск через bat (скопируйте [`frontend/electron/start-gis.bat.example`](frontend/electron/start-gis.bat.example) рядом с exe, переименуйте в `start-gis.bat`, исправьте IP, запускайте **bat**, не exe):

```bat
set GIS_API_URL=http://192.168.1.177:4000
GIS VOLS.exe
```

5. Или конфиг в `%APPDATA%\GIS VOLS\gis-desktop.json` — скрипт [`scripts/windows-create-desktop-config.bat`](scripts/windows-create-desktop-config.bat).

После успеха на экране входа: **Сервер API: http://192.168.1.177:4000** (не localhost).

#### Надёжный способ (в новой сборке): адрес на экране входа

На экране **Вход** в desktop-приложении есть поле **«Адрес сервера API (Mac)»**:

1. Введите `http://IP_ВАШЕГО_MAC:4000` (как в браузере, где `/health` открывается).
2. Нажмите **«Проверить связь»** — должно быть «Связь OK».
3. Войдите логином/паролем.

Адрес сохраняется на этом ПК в localStorage и **важнее** `gis-desktop.json`, если json не подхватился.

### 5. Автозапуск (Linux, systemd)

Файл `/etc/systemd/system/gis-backend.service`:

```ini
[Unit]
Description=GIS VOLS Backend
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/gis/backend
Environment=JWT_SECRET=ЗАМЕНИТЕ
Environment=PORT=4000
ExecStart=/usr/bin/node src/server.js
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now gis-backend
```

### 6. Данные и первый вход

Все базы: **`backend/data/`** (регулярно копируйте на NAS).

Первый пользователь (если база пустая): логин **Админ**, пароль **Админ** — сразу смените.

### 7. На сервере frontend не нужен

Для схемы «приложение на ПК» **не собирайте** сайт на сервере. Достаточно работающего backend.

# Часть 2. Собрать установщик приложения Windows (.exe)

Сборку удобно делать **на Windows** (тот же ПК или CI).

### 1. Установить Node.js LTS

https://nodejs.org/

### 2. Зависимости frontend

```bat
cd C:\GIS\frontend
npm install
```

### 3. Собрать приложение для Windows

**Вариант A — установщик (.exe Setup)** — нужен доступ к GitHub (скачивается NSIS ~2 МБ):

```bat
cd C:\GIS\frontend
npm run build:desktop
```

Результат: `frontend\release\GIS-VOLS-Setup-1.0.0.exe`

**Вариант B — без NSIS** (если установщик падает на `read tcp ... release-assets.githubusercontent.com` / таймаут сети):

```bat
npm run build:desktop:dir
```

Результат: папка `frontend\release\win-unpacked\` — скопируйте её на ПК пользователей, запуск `GIS VOLS.exe`. Рядом положите `gis-desktop.json` с `apiUrl`.

**Вариант C — ZIP-архив** (тоже без NSIS):

```bat
npm run build:desktop:zip
```

Результат: `frontend\release\GIS VOLS-1.0.0-win.zip` (имя может слегка отличаться по версии).

Первая сборка скачает Electron (~150 МБ) — нужен интернет. Если **`build:desktop` упал на шаге NSIS**, но до этого было `packaging ... appOutDir=release\win-unpacked` — приложение **уже собрано** в `release\win-unpacked`, можно раздавать папку без повторной загрузки Electron.

**Сборка с уже зашитым адресом сервера** (без json у каждого клиента):

```bat
set VITE_API_URL=http://192.168.1.50:4000
npm run build:desktop
```

Именно **`VITE_API_URL`** (не `VITE_API_URP`) и **`http://`** (не `htpp://`). Переменная должна быть задана **в той же** сессии cmd/PowerShell до `npm run build:desktop`.

### 4. Проверка без установщика

```bat
npm run build:desktop:dir
cd release\win-unpacked
GIS VOLS.exe
```

# Часть 3. Настроить приложение на ПК пользователя

После установки **GIS VOLS** нужно указать адрес сервера.

### Файл `gis-desktop.json`

```json
{
  "apiUrl": "http://192.168.1.50:4000"
}
```

Подставьте **IP или имя** вашего сервера (без слэша в конце).

### Куда положить файл (проверяется по порядку)

1. Рядом с `GIS VOLS.exe` (портативная папка)  
2. `%APPDATA%\GIS VOLS\gis-desktop.json` — **удобно для установщика NSIS**  
3. В папке ресурсов приложения (для продвинутых сценариев)

### Быстро создать конфиг (Windows)

Запустите от администратора:

`scripts\windows-create-desktop-config.bat`

Откроется блокнот с шаблоном — замените IP, сохраните.

### Запуск

Ярлык **GIS VOLS** на рабочем столе. Внизу окна в строке состояния: **Сервер API http://…** — проверьте, что адрес верный.

Войти логином/паролем → **Настройки** → выбрать рабочую базу (например «юл-ком»).

# Часть 4. Режим разработчика (Electron + локальный backend)

Для программистов на одном ПК:

```bat
cd backend
npm install
npm start
```

Второе окно:

```bat
cd frontend
npm install
npm run dev:desktop
```

Откроется окно; API по умолчанию `http://localhost:4000`, локальный backend стартует сам.

Подключение к **удалённому** серверу без локального backend:

```bat
set GIS_API_URL=http://192.168.1.50:4000
set GIS_START_LOCAL_BACKEND=0
npm run dev:desktop
```

# Часть 5. Альтернатива: браузер вместо приложения

Если .exe не используете:

1. На сервере или отдельно соберите сайт: `cd frontend`, `set VITE_API_URL=http://IP:4000`, `npm run build`.  
2. Раздайте папку `dist` через nginx/IIS.  
3. Пользователи открывают `http://сервер` в Chrome/Edge.

Для офиса обычно удобнее **приложение** (одна иконка, не путают вкладки).

# Часть 6. Частые проблемы

| Симптом | Решение |
|---------|---------|
| Белое окно / не грузится | Проверьте `gis-desktop.json`, перезапустите приложение |
| «Нужен вход» / ошибка сети | Запущен ли backend; пинг `http://IP:4000/health`; файрвол 4000 |
| Неверный сервер внизу окна | Исправьте `apiUrl` в `%APPDATA%\GIS VOLS\gis-desktop.json` |
| `npm install` в backend падает | Visual Studio Build Tools (C++) на Windows |
| Сборка .exe не на Windows | `npm run build:desktop` делайте на Windows (или CI windows-latest) |
| `Cannot create symbolic link` при `electron-builder` | В проекте отключена подпись (`CSC_IDENTITY_AUTO_DISCOVERY=false`). Обновите `frontend/package.json` с репозитория и снова `npm run build:desktop`. Либо: **Параметры Windows → Для разработчиков → Режим разработчика** (разрешает symlink), или cmd **от администратора** |
| `Electron failed to install correctly` | `rmdir /s /q node_modules\electron` затем `npm install`. Нужен интернет; антивирус не должен блокировать скачивание Electron |
| `dev:desktop` падает с той же ошибкой Electron | То же: переустановить `node_modules\electron`. На Windows для разработки backend обычно на Mac — `GIS_START_LOCAL_BACKEND=0` и `gis-desktop.json` с IP сервера |
| Таймаут при скачивании `nsis-3.0.4.1.7z` / `release-assets.githubusercontent.com` | Файрвол/прокси блокирует GitHub. Используйте **`npm run build:desktop:dir`** или **`build:desktop:zip`**, либо повторите `build:desktop` с VPN/другой сети. Готовая папка: `release\win-unpacked` |
| `Failed to fetch` / `/health` не открывается с Windows | Сначала `/health` **на Mac по IP Mac**, не `192.168.1.177` «наугад». Перезапуск `npm start`, файрвол macOS, один Wi‑Fi с Windows. `gis-desktop.json` с **тем же** IP, что в браузере |
